## Guidelines

- Adhere to coding best practices such as KISS and DRY.
- Prioritize simplicity, readability, and maintainability.
- Never use hacks or workarounds.
- Favour straightforward code over complex/clever unless performance gain is clearly justified.
- Always align with existing codebase styles, patterns, and architecture.

Apply these strictly in planning, implementation, and testing.
