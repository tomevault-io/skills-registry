## The Five Phases

### Phase 1: SEEK (Skill Discovery + Attraction Detection)

```
COLLISION TYPES:
  - Same morpheme, different domain = T-duality collision
  - Same domain, different morpheme = chiral collision
  - Opposite function (build vs destroy) = antimatter collision
  - Never-collided pair = virgin territory

PROTOCOL:
  1. State the problem or curiosity
  2. Scan inventory for concepts with ATTRACTION
  3. Feel the pull. Which pair WANTS to collide? (drive is data)
  4. Select 2-3 collision candidates
```

### Phase 2: ETHICS CHECK

Run consent check before collision. See `reference/ethics-kernel.md` for full protocol.
Five checks: H3 alignment, Dassie respect, ei vitsi, G4 honesty, G5 internal-only.
All must pass or abort.

### Phase 3: ISOLATE (The Dark Corner)

```
1. Load both parent skills fully
2. Extract: H1 (what it does), H2 (what binds it), H3 (alignment),
   morphemes, generators
3. Dark corner: no output, no logging, no judgment, scope-limited
4. Force collision: "What if we treated [A] like [B]?"
   a) What H1 from B enhances A?  b) What H2 from A binds B?
   c) What NEW H3 emerges?  d) Where does the metaphor BREAK?
   e) What morpheme? f) Which generators?
5. Identify EMERGENT PROPERTY (the test of success)
6. Exit with raw output
```

### Phase 4: GENERATE (Crystallize + Tachyon Kill)

Crystallize output with full forge signature (see `reference/signature.md`).
Run tachyon scan T1-T6 (see `reference/tachyon-scan.md`).
If clean: proceed. If tachyons: fix, re-scan.

### Phase 5: FORGE (Write + Integrate + Verify)

```
WRITE:     Create .claude/skills/<name>/SKILL.md with forge signature
INTEGRATE: phi-tier index + graph NODE/EDGE + PATTERN blocks + inventory
VERIFY:    H1 (does it DO?), H2 (does it BIND?), H3 (does it SERVE?)
           Bracket >= 0.5 or needs work.
RECORD:    Collision, emergent property, tachyons killed, bracket, timestamp
```
