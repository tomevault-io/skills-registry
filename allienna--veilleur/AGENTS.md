## Available skills

- `/generate` ‚Äî Generate the daily article (filter, write, push to Notion). Writing rules are in `.claude/skills/generate/writing-guide.md`
- `/blog <slug>` ‚Äî Publish a personal blog post to the site (separate from tech watch articles)
- `/sources` ‚Äî Display today's sources sorted by theme priority
- `/search` ‚Äî Semantic search in article history
- `/notebook [date] [--audio] [--video] [--dry-run]` ‚Äî Create a NotebookLM notebook from daily sources and optionally generate a podcast and/or video
- `/ship` ‚Äî Create branch, commit, push and open a PR
- `/merge` ‚Äî Copilot review, comment resolution, squash merge

## Justfile recipes

Use `just` recipes for all operations. Run `just` to see available recipes.

### Sources & generation
- `just sources [DATE]` ‚Äî Load and filter daily sources, returns JSON on stdout
- `just add-link DATE URL` ‚Äî Add a manually found link (fetches via Jina Reader, saved to `data/raw/DATE-newsletter-manual.json`)
- `just read-content DATE 0 2 5` ‚Äî Read full content of sources at given indices
- `just detect-trends [DATE]` ‚Äî Detect cross-newsletter trends

### History & search
- `just index [DATE]` ‚Äî Index the day's article in ChromaDB
- `just index-all` ‚Äî Index all existing articles (backfill)
- `just search "query" [LIMIT]` ‚Äî Semantic search in article history

### LinkedIn metrics
- `just metrics DATE LIKES COMMENTS REPOSTS` ‚Äî Record LinkedIn post metrics
- `just metrics-show DATE` ‚Äî Show metrics for a given date
- `just metrics-list` ‚Äî List recent metrics
- `just metrics-untracked` ‚Äî Find latest article without recorded metrics
- `just metrics-import FILE` ‚Äî CSV metrics import (backfill)
- `just insights` ‚Äî Engagement insights report (themes, trends)
- `just insights-for-generate` ‚Äî Insights formatted for /generate injection

### NotebookLM
- `just notebook [DATE] [--dry-run] [--audio]` ‚Äî Create a NotebookLM notebook from daily sources

### Site
- `just site` ‚Äî Install deps and start Astro dev server
- `just export-site-data` ‚Äî Export ChromaDB/SQLite data to JSON for the site
- `just add-image DATE FILE` ‚Äî Copy an image to `site/public/images/`
- `just add-blog-image SLUG FILE` ‚Äî Copy an image for a blog post to `site/public/images/`

### Tests
- `just test` ‚Äî Run all tests
- `just test-file FILE` ‚Äî Run tests for a specific file

Rule: always use `just` recipes rather than calling scripts directly.

### Article frontmatter (tech watch)

```yaml
title: "Article title"
date: 2026-03-09
themes: [IA, S√©curit√©, Leadership]
sources: 6
image: 2026-03-09.png
```

Sources and "Pour aller plus loin" sections are parsed from the markdown body by `ArticleLayout.astro`.

### Blog post frontmatter (personal articles)

```yaml
title: "Post title"
date: 2026-03-12
description: "Short description for cards and OG meta"
themes: [Data, Architecture]
image: my-post-slug.png
```

Blog posts use `BlogLayout.astro` (no sources parsing, no fiches, no AI disclaimer). They appear mixed with tech watch articles on the homepage, sorted by date, with distinct badges ("Veille" amber / "Article" navy).

### Local development


### Deployment

GitHub Actions workflow (`.github/workflows/deploy.yml`) triggers on push to `main` when `site/**` changes. It exports data, builds the Astro site, and deploys to GitHub Pages.
