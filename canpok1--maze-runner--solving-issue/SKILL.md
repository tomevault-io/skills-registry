## 手順

このチェックリストをコピーし、進行状況の追跡に使用してください：

タスク進捗：
- [ ] ステップ1：Issue番号を確認する
- [ ] ステップ2：既存PRの確認を行う
- [ ] ステップ3：`/coding` コマンドの手順を実行する（実装・テスト）
- [ ] ステップ4：`/reviewing` コマンドの手順を実行する（自己レビュー）
- [ ] ステップ5：gitコマンドで変更をコミット・プッシュする
- [ ] ステップ6：`/creating-pr` コマンドの手順を実行する（PR作成）
- [ ] ステップ7：`/fixing-pr` コマンドを実行してPRの修正・マージまで完了させる

1. Issue番号を確認する。
    - 引数で指定されている場合、その番号を使用する。
    - 引数が未指定の場合は「GitHub Issueの番号を指定してください」と通知し、処理を終了する。
2. 既存PRの確認を行う。
    - Issue番号に対応するopen状態のPRを検索。複数あれば最も新しいものを選択。
    - PRが見つかれば手順7へ、なければ手順3へ。
3. `/coding` コマンドを実行（実装・テスト）。
    - 完了後、ユーザーに「codingスキルが完了しました。次にreviewingスキルを実行します。」と報告する。
4. `/reviewing` コマンドを実行（自己レビュー）。
    - 完了後、ユーザーに「reviewingスキルが完了しました。次にコミット・プッシュを実行します。」と報告する。
5. gitコマンドで変更をコミット・プッシュする（CLAUDE.mdのコミット規則に従う）。
    - 完了後、ユーザーに「コミット・プッシュが完了しました。次にcreating-prスキルを実行します。」と報告する。
6. `/creating-pr` コマンドを実行（PR作成）。
    - 完了後、ユーザーに「creating-prスキルが完了しました。次にfixing-prスキルを実行します。」と報告する。
7. `/fixing-pr` コマンドを実行してPRの修正・マージまで完了させる。
    - fixing-prがPRのマージに成功した場合は処理完了。
    - fixing-prがマージを実行できなかった場合（承認待ち等の外的要因）は、作業完了として処理を終了する。
    - PRがマージされずにクローズされた場合は作業中断。

## エージェント利用ルール

- ソースコードの実装は `coding-specialist` エージェントに依頼すること
    - 可能な限り複数エージェントで並列実行すること
    - 複数エージェントが作業する際は次の作業などで競合するため注意
        - 品質チェック手順の実行
        - 同一ファイルの編集
- 文章（プルリクエストやタスクの内容など）を考える際は `document-specialist` エージェントに依頼すること
- スキル（`.claude/skills/` 配下の定義）の作成・レビュー・改善は `skill-specialist` エージェントに依頼すること

## 注意点
- エラー発生時は報告し作業を中断する。
