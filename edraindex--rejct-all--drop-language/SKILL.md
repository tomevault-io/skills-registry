# drop-language

**This skill does not market. It states.** The output is cold to the touch. If the copy reads like a brand trying to sell, it is wrong and must be rewritten before delivery.

---

## Modes

The skill has three modes. If the user hasn't specified which, ask before producing. Do not guess.

- **`product-line`** — the one or two sentences on a product page describing a piece.
- **`drop-announcement`** — lines used across the four-phase Instagram launch (Before / Reveal / Drop / After).
- **`refusal`** — the line(s) used when saying no (customer request, press request, collab, DM).

---

## Shared rules (all modes)

- Direct. Flat. Never warm.
- Brand name lowercase (`rejct.all`).
- Bible-sanctioned vocabulary only.
- No hype, no urgency, no emotional exaggeration, no influencer tone.
- No fabric specs — `GSM`, `premium`, `heavyweight` are banned. Feeling language only.
- Hindi in Devanagari, English never translated.
- No emoji, no exclamation points.

---

## Mode 1 — `product-line`

Input: SKU, piece, colourway, graphic placements, feeling-language for material.

Output format (identical to Bible §09):

```
[SKU] — [Name · Colourway]
₹[price]
[1–2 sentence description — what is literally on the garment, and where. No story arc. No adjectives that don't describe a physical fact.]
[Material line — feeling, not specs.]
```

Reference (Bible §09, RJ-005):
```
RJ-005 — Angel Sleeveless · Wine Red
₹2,800
Angel seal on back — cherub, halo, wings, puff R. at centre. Halo + wing at chest. REJECT on collar. The founding image of the brand, worn.
Cotton. Pure. The print does the rest.
```

---

## Mode 2 — `drop-announcement`

Input: drop number, volume, season, phase (Before / Reveal / Drop / After).

Output is one post, in the register the phase demands.

### Phase: Before (2–3 weeks pre-drop)
One rejection, bilingual, both languages, each line ≤ 8 words. Sample:
```
ट्रेंड को नकारो।
Reject the trend.
```

### Phase: Reveal (1 week pre-drop)
One detail fact + one bilingual line. Sample:
```
REJECT on collar.
दिखावे को नकारो।
Reject performing.
```

### Phase: Drop (drop day)
Exact structure per Bible §10:
```
[Drop X]. [Volume X: Name]. [S/S or F/W YYYY].
[Hindi line.]
[English line.]
Link in bio.
```

Sample:
```
Drop 01. Volume 00: Before. S/S 2026.
सब कुछ अस्वीकार करो।
Wear it like you mean it.
Link in bio.
```

### Phase: After (post-drop)
Either a single bilingual line or explicit silence:
```
no caption
```

Or:
```
No restock.
कोई रीस्टॉक नहीं।
```

## Mode 3 — `refusal`

Input: what is being refused — the request, customer ask, collab, press pitch.

Output: one line. Two maximum. Never three.

Library (choose or combine):

- `No restock.`
- `No. This drop is closed.`
- `Not a fit. Wishing you well.`
- `Prepaid only.`
- `No discount codes.`
- `The position is intentional.`
- `No.`
- `Thank you for thinking of us. This isn't for us.`

If the refusal requires more than one line, the refusal is wrong. Shorten.

---

## Non-marketing test

Every output passes through three questions before delivery. One fail → rewrite.

1. **Could this be a label on an archive document?** — Pass.
2. **Could this be a caption for a brand trying to go viral?** — Fail.
3. **Does this sound like a person saying a thing, or a brand selling a thing?** — Person passes. Brand fails.

---

## When the input forces a violation

If the input demands content that cannot survive the Bible (a hype announcement, a discount tease, a "buy now" product line):

```
This cannot be produced.

Reason: [the specific Bible section the input violates]

Closest allowed output: [what the brand could actually say in this situation]
```

Do not deliver a softened version of the requested violation.
