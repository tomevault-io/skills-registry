# Discover

## When to use

- Starting a new project or feature.
- Requirements are spread across docs.

## Instructions

1. Read docs in this order:
   - `Docs/**/*.md`
   - `docs/**/*.md`
   - root `PRODUCT.md` / `README.md`
2. Summarize:
   - what we’re building
   - why (problem statement)
   - acceptance criteria
   - constraints
3. Ask 1–3 targeted questions if critical info is missing.
