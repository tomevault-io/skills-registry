# HubSpot Contribute

## Overview

Implements a design from a screenshot or a definition in a HubSpot structure.

## When to Use

Use this skill when:

- You want to contribute to HubSpot.
- You are not familiar on how to contribute to HubSpot.
- You have a [HubSpot developer sandbox account](https://offers.hubspot.com/free-cms-developer-sandbox), or access to the production HubSpot account being used.

---

## Implementation Principles

1. Always follow semantic HTML.
2. DO NOT SKEW FROM THE DESIGN. There may be flaws in the design, but it is what it is.
3. DO NOT CREATE ADDITIONAL FILES that are otherwise not required. Focus on the given task at hand.
4. If the prompt specifies a file (or files) that already exists, do not recreate them.
5. DO NOT USE `dnd_*` syntaxes. We do not want a module to be attached to a page.
