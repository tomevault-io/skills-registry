# TDD（テスト駆動開発）

## ワークフロー

1. **Red**: 失敗するテストを書く
2. **Green**: テストを通す最小限の実装
3. **Refactor**: コードを整理

## テスト作成ルール

- table driven testパターンを使用
- テストケース名は日本語で記述
- 境界値・異常系を含める
