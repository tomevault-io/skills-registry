## What I do

- Structure Vue.js screens for analytics-heavy use cases.
- Define how filters, summaries, tables, and charts should work together.
- Keep visual hierarchy clear when many market signals compete for attention.
- Call out loading, empty, stale, and error states.

## Default patterns

- Top-level KPI summary row for the current filter context.
- Persistent filter panel or filter bar.
- Main analytical canvas with trend and composition charts.
- Drill-down table for vacancy-level or grouped detail.
- Clear freshness indicator when realtime or near-realtime data is shown.

## Guardrails

- Avoid generic admin-panel layouts when the data story needs stronger focus.
- Do not hide metric definitions behind vague labels.
