# Plan Review Skill

## Overview

This skill guides plan reviews according to Abonmarche standards. Each review stage has specific checks appropriate to the design completion level.

## Determine Review Type

Ask the user which review stage if not specified:

- **30% Review**: Conceptual/schematic design validation
- **60% Review**: Design development verification
- **90% Review**: Final/construction document review

## Review Workflows

**For 30% Review**: See [reviews/30-percent.md](reviews/30-percent.md)

**For 60% Review**: See [reviews/60-percent.md](reviews/60-percent.md)

**For 90% Review**: See [reviews/90-percent.md](reviews/90-percent.md)

## Review Process

1. Confirm review stage with user
2. Load the appropriate review checklist
3. Work through each check item systematically
4. Document findings using the output format below
5. Summarize issues by severity

## Output Format

For each issue found:

```
## [Check Category]

### Issue: [Brief description]
- **Severity**: Critical / Major / Minor
- **Location**: [Sheet/Area reference]
- **Finding**: [Detailed description]
- **Recommendation**: [Suggested resolution]
```
