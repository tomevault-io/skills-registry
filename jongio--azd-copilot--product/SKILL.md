# Product Skills

Skills for defining product requirements and managing the product lifecycle on Azure.

## Available Skills

| Skill | Description | When to Use |
|-------|-------------|-------------|
| [requirements](requirements.md) | Gather and document requirements | Starting new features |
| [acceptance-criteria](acceptance-criteria.md) | Define testable acceptance criteria | Before development |
| [user-stories](user-stories.md) | Create user stories | Backlog grooming |
| [azure-capabilities](azure-capabilities.md) | Azure service capability matrix | Evaluating feasibility |

## Azure Specialization

The product role has deep knowledge of Azure capabilities and limitations:

- **Service Limits**: Know quotas, rate limits, regional availability
- **Pricing Models**: Understand consumption vs provisioned pricing
- **Feature Availability**: Track preview vs GA features
- **Compliance**: Understand Azure compliance certifications

## Quick Reference

### Requirements Gathering

1. Identify stakeholders
2. Define user personas
3. Document functional requirements
4. Document non-functional requirements (performance, security, compliance)
5. Validate against Azure capabilities

### User Story Format

```
As a [persona],
I want to [action],
So that [benefit].

Acceptance Criteria:
- Given [context]
- When [action]
- Then [expected result]
```
