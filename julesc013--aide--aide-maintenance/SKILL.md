## Scope

- `MAINTENANCE.md`
- `scripts/maintenance/**`
- root-doc and control-plane sync work
- maintenance-oriented audit and upkeep tasks

## Trigger

- when a prompt asks for repository maintenance, upkeep, sync review, or operational hygiene
- when maintenance catalogs, checklists, or recurring review assets need updates
- when blocked, deferred, or candidate posture must be kept explicit across docs

## Do Not Use

- Do not use this skill for product implementation work.
- Do not use this skill when the task is only host-specific code or research with no maintenance component.
