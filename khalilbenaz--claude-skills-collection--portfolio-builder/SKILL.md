# Portfolio Builder

## Workflow
1. **Définition de l'objectif** — Clarifier l'objectif du portfolio : décrocher des missions freelance, trouver un CDI, montrer une expertise spécifique, ou construire une marque personnelle.
2. **Sélection des projets** — Choisir 4-8 projets représentatifs selon des critères : pertinence par rapport à la cible, diversité des compétences démontrées, résultats mesurables, qualité visuelle.
3. **Storytelling par projet** — Pour chaque projet, structurer la présentation : contexte et problème client, solution technique choisie (et pourquoi), défis rencontrés et solutions apportées, résultats concrets (métriques, impact business), stack technique utilisée.
4. **Page À Propos** — Rédiger une page personnelle engageante : parcours professionnel, valeurs, spécialités, ce qui vous différencie, photo professionnelle.
5. **Structure du site** — Organiser le portfolio : page d'accueil avec proposition de valeur claire, galerie de projets filtrable, page à propos, témoignages clients, page contact avec CTA visible.
6. **Optimisation technique** — Assurer : responsive design, temps de chargement rapide (< 3s), SEO (balises meta, schema.org, sitemap), accessibilité (WCAG 2.1), analytics (Google Analytics ou Plausible).
7. **Choix de la plateforme** — Recommander une solution adaptée : site custom (Next.js, Astro), plateforme dédiée (Behance, Dribbble), constructeur (Webflow, Framer), ou GitHub Pages pour les développeurs.
8. **Maintenance et mise à jour** — Planifier des mises à jour trimestrielles : ajouter les nouveaux projets, retirer les projets obsolètes, actualiser les témoignages et les métriques.

## Règles
- Qualité plutôt que quantité : 5 projets bien présentés valent mieux que 20 projets bâclés.
- Chaque projet doit montrer un résultat concret, pas juste une liste de technologies.
- Ne jamais publier de code ou d'informations client sans autorisation explicite.
- Le portfolio doit être accessible et rapide sur mobile (plus de 60% du trafic).
