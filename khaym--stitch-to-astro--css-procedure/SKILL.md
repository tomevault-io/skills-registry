# Stitch to CSS Procedure

Step-by-step procedure for transforming cached Stitch design data into CSS Custom Properties (design tokens) + CSS reset in `src/styles/global.css`.

## Prerequisites

- `docs/stitch/reference.md` exists (generated by stitch-cache skill)
- User's design decisions are recorded in `progress.md` notes

## Token Count Guideline

Keep the total token count minimal. Reference values from the travellog implementation:

| Category | Target count | Notes |
|---|---|---|
| Color | ~18 | Neutral 9, Accent 3, Semantic 3, Link 3 |
| Font family | 3 | serif, sans, mono |
| Font weight | 2-4 | normal, bold + optional light, semibold |
| Font size | 6-8 | caption through display. Only sizes actually used in design |
| Line height | 5-7 | DADS levels used in the design |
| Letter spacing | 3 | tight, normal, wide |
| Spacing | 8-10 | 8px base unit multipliers used in the design |
| Corner shapes | 5 | none, sm, md, lg, full |
| Elevation | 6-7 | Shadow levels + overlay shade |
| Layout | 3 | max-width, grid-columns, grid-gutter |
| **Total** | **~60-70** | |

**Principle:** Define only tokens that the design actually uses. Do not pre-generate a complete matrix of all possible values.

## Steps

### Step 1: Extract and Unify Color Tokens

#### 1-1. Collect Colors from Stitch

Refer to the color tables in `reference.md` for each screen.
The `tailwind.config` `colors` object in Stitch HTML is the primary source.

#### 1-2. Apply User Decisions

When screens use different values for the same purpose, follow decisions recorded in `progress.md` notes.
If no decision exists, ask the user.

#### 1-3. Compose Color Tokens per DADS System

Follow design-guideline's "1. Color" section:

| Token Category | Source | Method |
|---|---|---|
| Neutrals (gray scale) | Stitch zinc/gray values | Reference DADS contrast ratios (Gray-420, Gray-536, Gray-600); compose ~10 steps |
| Accent | User-unified value | Base color + light/dark variations |
| Semantic | DADS-compliant | success (green), error (red), warning (yellow/orange) |
| Link | DADS-compliant | default (blue), visited (purple), active (orange) |

### Step 1.5: Resolving DADS vs Stitch Differences

When Stitch values differ from DADS token system, use this decision flow:

```
Compare Stitch value with DADS value
  ‚îÇ
  ‚îú‚îÄ Match ‚Üí Use as-is
  ‚îÇ
  ‚îú‚îÄ Small difference ‚Üí Align to one side (do NOT add variant)
  ‚îÇ    ‚îÇ
  ‚îÇ    ‚îú‚îÄ Functionally meaningful difference?
  ‚îÇ    ‚îÇ    Yes ‚Üí Use Stitch value (respect design intent)
  ‚îÇ    ‚îÇ    No  ‚Üí Use DADS value (maintain system consistency)
  ‚îÇ    ‚îÇ
  ‚îÇ    ‚îî‚îÄ Uncertain ‚Üí Default to DADS
  ‚îÇ
  ‚îî‚îÄ Large difference ‚Üí Add new token with Stitch value
```

#### "Small difference" thresholds

| Category | Threshold | Example |
|---|---|---|
| Color | Visually near-identical hue/lightness | Stitch `#71717a` vs DADS Gray-536 equivalent |
| Font size | ¬±2px | Stitch `15px` ‚Üí align to DADS `14px` or `16px` |
| Spacing | ¬±4px (half base unit) | Stitch `28px` ‚Üí align to DADS `24px` (3x) or `32px` (4x) |
| Border radius | ¬±4px | Stitch `10px` ‚Üí align to DADS `8px` (sm) or `12px` (md) |

#### Principles

- **Keep token count low.** Do not add variants for minor differences
- **Default to DADS.** When uncertain, use DADS values
- **Respect clear Stitch design intent** (e.g., specific accent color, distinctive weight)

---

### Step 2: Define Typography Tokens

#### 2-1. Determine Font Families

Check typography tables in `reference.md`, unify per user decisions.

#### 2-2. Define Tokens per DADS System

Follow design-guideline's "2. Typography" section:

| Token | Method |
|---|---|
| Font family | User-decided value. 3 families: serif / sans / mono |
| Font weight | DADS base: normal(400), bold(700). Add weights used in Stitch as needed |
| Font size | Only sizes that appear in the Stitch design, mapped to DADS bands (Display/Standard/Dense/Caption) |
| Line height | DADS levels (100%-175%) as-is |
| Letter spacing | DADS: tight(-0.01em) / normal(0) / wide(0.02em) |

---

### Step 3: Spacing, Border Radius, Elevation

These are largely Stitch-independent; adopt DADS system directly.

| Category | Source | Method |
|---|---|---|
| Spacing | DADS "6. Spacing" | 8px base unit, multiplier scale (8px-128px) |
| Border radius | DADS "7. Corner Shapes" | 5 levels: none / sm(8px) / md(12px) / lg(16px) / full |
| Elevation | DADS "8. Elevation" | 6 practical drop-shadow levels + overlay shade |

---

### Step 4: Layout Tokens

Combine `reference.md` layout info with DADS "4. Layout":

| Token | Method |
|---|---|
| `--content-max-width` | Determine from Stitch device width and content area |
| `--grid-columns` | DADS: 12 columns |
| `--grid-gutter` | DADS: 2x body font size |

---

### Step 5: CSS Reset

Add a modern CSS reset at the top of `global.css`, referencing design tokens:

```css
*, *::before, *::after { box-sizing: border-box; }
* { margin: 0; padding: 0; }
html { -webkit-text-size-adjust: 100%; tab-size: 4; }
body {
  min-height: 100dvh;
  line-height: var(--line-height-normal);
  font-family: var(--font-sans);
  font-size: var(--font-size-base);
  font-feature-settings: "palt";  /* Japanese proportional metrics */
}
img, picture, video, canvas, svg { display: block; max-width: 100%; }
input, button, textarea, select { font: inherit; }
h1, h2, h3, h4, h5, h6 { overflow-wrap: break-word; text-wrap: balance; }
p { overflow-wrap: break-word; text-wrap: pretty; }
```

---

### Step 6: Base Element Styles

Define link states per design-guideline "5. Link Text":

```css
a          { color: var(--color-link);         text-decoration: underline; }
a:visited  { color: var(--color-link-visited); }
a:hover    { /* lighter blue + thicker underline. No size change (DADS) */ }
a:active   { color: var(--color-link-active); }
```

Also define `::selection` with the accent color.

---

### Step 7: Load in BaseLayout

Import `global.css` in the common Astro layout:

```astro
---
import "../styles/global.css";
---
```

Load any web fonts decided by the user via `<link>`:

```html
<link href="https://fonts.googleapis.com/css2?family=<Font+Name>:wght@400;600&display=swap"
      rel="stylesheet" />
```

## Reference: Stitch (Tailwind) ‚Üí CSS Custom Properties Conversion

| Stitch (Tailwind) | global.css (CSS Custom Properties) |
|---|---|
| `tailwind.config.colors.accent: "#xxx"` | `--color-accent: #xxx;` |
| `text-zinc-700` class | Define as `--color-gray-700: #xxx;`, reference in component CSS |
| `font-serif: ["Font Name"]` | `--font-serif: "Font Name", serif;` |
| `text-[10rem]`, `text-4xl` etc. | Map to DADS scale as `--font-size-*` tokens |
| `py-48`, `gap-24` etc. | Map to 8px-based `--spacing-*` tokens |
| Tailwind implicit reset (`@layer base`) | Explicit CSS reset rules |
