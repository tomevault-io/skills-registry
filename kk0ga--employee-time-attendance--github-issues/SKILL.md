# GitHub Issues Skill

## 目的
小規模でも「口頭/チャットの依頼」を消さずに、
Issueで **要件・受け入れ条件・作業範囲** を固定する。

## 書き方（推奨）
### バグ
- 現象
- 再現手順
- 期待する動作 / 実際の動作
- 環境（ブラウザ/OS/バージョン）

### 機能
- 背景（なぜ必要か）
- 仕様（画面・入力・権限）
- 受け入れ条件（テスト可能な形）

## 依頼例
- 「ログイン失敗のIssueを作って。再現手順と期待値も入れて」
- 「勤怠のPDF出力を機能Issueとして整理して」
