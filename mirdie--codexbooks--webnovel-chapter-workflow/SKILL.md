# Webnovel Chapter Workflow

## Overview

把“写一章”拆成可执行步骤，保证节奏、信息量与钩子稳定。

## Workflow

1. 章节目标设定
   - 章节目的：推进主线 / 人物变化 / 设定落地（可多选）
   - 章节变化：读者读完后“知道了什么/发生了什么”

2. 冲突与悬念
   - 本章主冲突：目标 vs 阻力
   - 结尾钩子：信息反转/新威胁/未完成动作

3. 节奏分配（四拍）
   - 开场钩子：快速进入情境
   - 推进段：行动或对话推进
   - 转折段：信息变化或对立升级
   - 结尾钩子：制造下一章需求

4. 信息密度控制
   - 每章至少 1 条关键新信息
   - 避免“原地踏步”或重复解释

5. 修订检查（不改写正文，只提问题）
   - 视角是否稳定
   - 冲突是否明确
   - 节奏是否拖沓
   - 钩子是否有效

## Output Format

只输出章节提纲或结构，不写正文：

```
章节：ch-0012
目标：
主冲突：
关键变化：
四拍：
  - 开场钩子：
  - 推进段：
  - 转折段：
  - 结尾钩子：
```

若用户要求正文，先确认字数上限与视角，再按小段输出。
