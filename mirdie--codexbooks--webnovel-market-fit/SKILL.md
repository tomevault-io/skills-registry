# Webnovel Market Fit

## Overview

用“平台—题材—读者需求”三角定位，降低选题失配风险。

## Workflow

1. 平台与规则梳理（只在用户提供时使用）
   - 平台调性、审核红线、分成规则（不臆测）

2. 题材定位
   - 题材赛道：玄幻/都市/悬疑/历史等
   - 子类型标签：复仇/系统/灵异等（仅标签层）

3. 竞品区间定位（不点名作品）
   - 热门方向：读者强需求点
   - 红海风险：同质化密集区域
   - 蓝海机会：稀缺但可被理解的卖点

4. 卖点承诺校准
   - 3 条核心卖点（不写剧情）
   - 读者预期与兑现路径匹配

5. 风险检查
   - 与平台红线冲突
   - 题材与受众错配
   - 卖点过度抽象或不可持续

## Output Format

```
平台：待确认
题材赛道：
子类型标签：
热门方向：
红海风险：
蓝海机会：
核心卖点（3条）：
待确认项：
```
