# Webnovel Worldbuilding Bible

## Overview

把世界观从“灵感”转成可执行规则，保证长期连载不自相矛盾。

## Workflow

1. 设定范围控制
   - 只定义剧情需要的层级与区域，避免无用扩张

2. 规则系统（核心）
   - 规则：能做什么
   - 成本：代价/限制/风险
   - 例外：可被打破的条件与代价
   - 冲突触发点：哪些矛盾能由规则驱动

3. 势力与资源
   - 阵营清单：目标、资源、对立点
   - 资源流动：获取路径、稀缺度、冲突点

4. 时空与历史
   - 时间线：关键事件列表
   - 地理：必要地图或区域表（抽象即可）

5. 设定-剧情对齐
   - 新设定必须服务剧情或人物成长
   - 不为“炫设定”牺牲节奏

6. 变更策略
   - 任何设定修订需记录在 continuity-log.md
   - 遇到冲突先提出修订方案而非直接改写正文

## Output Format

只输出结构化设定，不写剧情段落。示例：

```
规则体系：能力X
规则：
成本：
例外：
冲突触发点：

势力：阵营A
目标：
资源：
冲突点：
```

信息不足时列“待确认项”，不擅自补剧情。
