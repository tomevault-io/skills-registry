# project-management

Use this skill for PMO 视角的项目管理：把目标拆解为可交付、可跟踪、可复盘的计划。

## Outputs (choose what the task needs)

- Project charter（目标/范围/成功标准/约束/资源）
- WBS（任务分解）+ milestones（里程碑）+ timeline
- RACI（责任矩阵）+ stakeholder map
- RAID log（Risks/Assumptions/Issues/Dependencies）
- Weekly status report（进度/风险/下一步/需要支持）
- Release plan（上线窗口/回滚/验证/沟通）

## Workflow

1) Define objective and success criteria
- 目标是什么？完成的定义（DoD）是什么？
- 关键指标与验收口径（可测试、可签字）。

2) Scope management
- In-scope / out-of-scope 明确写清。
- 变更流程：谁批准、怎么评估影响、如何同步。

3) Plan the work
- 拆解到可执行任务（1–3天粒度更好）。
- 明确依赖（跨团队/外部系统/资源）。
- 关键路径与缓冲（buffer）设置。

4) Assign responsibilities
- RACI：Responsible/Accountable/Consulted/Informed。
- 明确单点 owner，避免多人“共同负责”。

5) Track and communicate
- 每周固定节奏：同步进度、阻塞、风险、决策点。
- 用数据说话：完成率、燃尽、缺陷趋势、发布成功率。

6) Risk management (RAID)
- 风险：概率×影响，给出缓解/预案/触发条件。
- 问题：当前已发生，明确 owner 与截止时间。
- 依赖：确认交付物与日期，建立升级路径。

7) Release coordination
- 上线 checklist：变更范围、灰度策略、监控指标、回滚方案。
- 沟通计划：谁需要提前知道、上线时谁 on-call、事后复盘。
