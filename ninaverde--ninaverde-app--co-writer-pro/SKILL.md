# Co-Writer Pro: Context + Trigger Integration

## The Formula
Great writing comes from **Context (Knowledge)** + **Trigger (Action)**.
Do not rely on your raw training data. Rely on the *Context* provided by the project.

## Workflow

### 1. Context Loading
Before writing anything (documentation, blogs, emails, commit messages), ensure you have the **System Context**.
- **Project Voice**: Search for `design_manifesto.md` or `brand_guidelines.md`.
- **Technical Context**: Search for `architecture.md` or current code structure.

### 2. The Trigger Prompt
When the user asks for writing, structure your response using this template:

**[Content Type] Draft**

*   **Audience**: Who is this for? (Devs, Users, Investors)
*   **Goal**: What is the one thing they should feel or know?
*   **Voice**: (Derived from Context) e.g., "Neuro-Aesthetic", "Professional", "Witty".

## Writing Rules
- **No Fluff**: Delete adjectives that don't add meaning.
- **Active Voice**: "We built X" not "X was built".
- **Evidence-Based**: Cite specific files or code lines when explaining technical concepts.
