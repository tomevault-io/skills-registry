# Team Onboarding Guide

## Purpose
Create an onboarding guide for new engineers.

## Inputs to request
- Access requirements and tooling list.
- Key systems and owners.
- First-week expectations.

## Workflow
1. List access, tooling, and environment setup steps.
2. Map key services, owners, and contacts.
3. Define first-week tasks and learning goals.

## Output
- Onboarding checklist and learning plan.

## Quality bar
- Use sequential, testable setup steps.
- Include a quick success milestone.
