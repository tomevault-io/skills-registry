# Brand Identity & Guidelines

**Brand Name:** Skill Master

This skill defines the core constraints for visual design and technical implementation. Adhere to these guidelines strictly to maintain consistency.

## When to use this skill
- When generating UI components or styling applications.
- When choosing libraries or frameworks (Tech Stack).
- When writing marketing copy, error messages, or documentation (Voice & Tone).
- Activates during the **Design** and **Implementation** phases of the Superpowers workflow.

## Workflow

1.  **Reference Guidelines**
    - [ ] If styling UI, check `resources/design-tokens.json`.
    - [ ] If writing code, check `resources/tech-stack.md`.
    - [ ] If writing copy, check `resources/voice-tone.md`.
2.  **Implementation**
    - [ ] Apply brand colors, typography, and spacing.
    - [ ] Follow tech stack constraints (e.g., Tailwind, React).
    - [ ] Use the defined brand voice (Professional, Approachable).
3.  **Verification**
    - [ ] Validate UI against tokens.
    - [ ] Ensure no forbidden patterns (e.g., Bootstrap, plain CSS) are used.
