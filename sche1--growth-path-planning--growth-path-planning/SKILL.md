# 成长路径规划

把模糊的"我想成为什么 / 学什么 / 到达什么方向"——**尤其是同时背着几条看起来分散的线**（DJ + 炒股 + 玩具 + 数学很正常）——变成一张**揭示跨线共底层、有先后依赖、有高杠杆优先级**的攻坚地图。

## 核心卖点（这才是它存在的理由）

人是复杂的，分散是常态。表面看是几条不相干的线，**底层往往共享同一批原子，特别是心法**。本 skill 的真正价值是把多条线拼成一张共底层 DAG——那些"被多条线同时依赖的底层节点"自然浮成最高 ROI 攻坚点：**投一次，多条线一起前进**。

单方向规划也成立（少数线的特例），但本 skill 的锐利之处在多线整合。

## 何时用

- 用户给出**几条想推进的不同方向**，要"从哪下手、什么先学什么后学、能不能同时干"。
- 用户只有一条方向但内部很杂，也适用（多线退化为单线）。
- 输入可为任意形态：一段话、几条并列短语、草稿、或已有旧文档。

## 何时不用

- 用户只问单一具体知识点 → 直接回答即可。
- 用户要执行学习或追踪进度 → 不在范围（本 skill 只产规划）。

## 关键定义（锋利、可操作）

- **技能（hard skill）= 产出某项 X 的能力**。可考核：能不能做出 X。例：能做出一首可发布的混音、能写一份估值模型、能解一道概率题、能鉴定一款玩具真伪。**Output-capability。**
- **心法（mindset / 内功）= 基础模型 / 世界观先验**——看世界的底层透镜，决定所有技能怎么被理解和调用。深于技能、跨域复用。例：概率思维、第一性原理、长期主义、美学直觉、因果思维。**和 LLM 同构：心法是 pre-training，技能是 fine-tune。**
- **关系**：心法是技能的 prior。同一条心法（如"概率思维"）可被量化策略 / A/B 测试 / 扑克 / 风险管理多个技能共享调用——这是多线整合时"高杠杆节点"的真实来源。

## 核心流程（按序执行）

1. **吸收混合输入**：原样接收，不要求重写、不嫌乱。明确每条线（即便用户没分线，主动识别 2-N 条）。
2. **靶向补问**：只就影响拆解的关键缺口提 1–4 个问题——现有水平 / 每周可投入时间 / 硬约束 / 各线优先级。不从零问起。
3. **逐线双轨原子化**：对每条线独立拆出"技能（产出能力）+ 心法（基础模型）"原子，标重要性 + 直接前置。
4. **跨线共底层识别（核心步骤）**：扫一遍所有原子，**主动合并跨线共享的心法和技能**——同一条"概率思维"在炒股和数学下出现就合并为**一个节点**（不是两份）。这些合并节点是高杠杆候选。
5. **建依赖图**：mermaid `graph TD`，必须无环。跨线节点位于图中央，各线特有技能从它们发散出去。
6. **标高杠杆**：按出度（被直接依赖数）降序；**跨线共享节点天然排在前面**——这是核心卖点的实操体现。
7. **排攻坚顺序**：合法拓扑序前提下，高杠杆共享节点优先。一份投入回报多条线。
8. **出洞察**：「零门槛起手项」/「够不着、先放着的远期目标」。
9. **可选独立挑刺复核**：草稿完成后**默认询问用户是否启用**；用户确认才执行下面"独立挑刺复核"一节；不确认则草稿即定稿。
10. **定稿**：按"标准输出模板"产出单一规划文档。

## 双轨原子化规则（细化）

每个原子必标：

- **轨道**：技能 / 心法（按上面定义判，分歧时往**底层**靠——若它构成你看世界的透镜，归心法）。
- **重要性 ★1–5**。
- **直接前置依赖**：列前置原子名；无则写"—"。
- **服务的线**（多线场景必填）：列出此原子服务于哪几条线（用于步骤 4 的合并判定）。

拆到**最小可学单元**——再拆没有意义为止。

## 依赖图规则

- 输出 mermaid `graph TD`；箭头 `A --> B` 表示"A 是 B 的前置（先会 A 再学 B）"。
- **必须无环**：生成后自检；发现环则明确报告，并指出哪条依赖应删或反向以打破。
- 节点用短标签，旁注完整原子名。**跨线共享节点**用统一节点（合并后），并在节点旁注"服务: A线/B线/..."。

## 高杠杆识别

- 统计每个原子的出度。出度高 = 高杠杆地基项。
- **跨线合并节点的出度自然更高**（被多条线下游同时依赖），这是核心卖点的算术体现。
- 按出度降序列 Top 高杠杆项。在 mermaid 中给这些节点上色：`style 节点名 fill:#ff6b6b`。

## 攻坚顺序

- 在满足依赖（合法拓扑序）的前提下，**高杠杆共享项优先**——这条规则在多线场景下最锋利：先攻共享心法/技能，多条线同时往前。
- 切成阶段（如「地基期 / 展开期 / 产出期」），每阶段列：本阶段攻坚原子 + 完成判据。
- 显式声明：未满足前置的原子**不得**排进早期阶段。

## 可选：独立挑刺复核

草稿完成后问用户："要不要让一个**独立 AI** 专门挑刺复核这份规划？"

用户确认才执行：用 Agent 工具 spawn 一个**前台、非团队（不传 team_name）、不后台（不设 run_in_background）**的独立 subagent。其最终消息会直接作为结果返回——可靠，不走团队 mailbox，规避权限死锁。

> 你是独立挑刺复核员。**只用 Read / Grep 工具读取；禁止 Bash、禁止写文件、禁止网络、禁止 git。**
> 审查下面这份成长路径规划草案。逐条查：
> ① 技能 vs 心法判定是否符合定义（技能=产出能力 / 心法=基础模型）？有没有被错放轨的？
> ② 跨线共底层是否识别充分？有没有该合并而未合并的原子？
> ③ mermaid 是否**有环**或缺关键前置？
> ④ 高杠杆项识别是否漏掉真正的地基（尤其跨线共享节点）？
> ⑤ 攻坚顺序是否违反依赖？是否真的"投一次受益多线"？
> ⑥ 是否混入了用户没要的范围、或漏掉用户明确要的线？
> 输出缺陷清单，每条：`[严重度 高/中/低] 问题 | 证据 | 具体改法`。最后给总评判（可定稿 / 改"高"后定稿 / 需重拆）。只读，不改任何文件。

收到清单后，对"高 / 中"项做**一次**修订，再定稿。

## 标准输出模板（精核，固定五块）

产出单一 markdown 文档，依次含：

1. **原子盘点表** —— 两小表（技能 / 心法）；每表列：`原子 | 重要性★ | 前置依赖 | 服务的线`。
2. **mermaid 依赖关系图** —— 无环；跨线共享节点居中；高杠杆节点上色。
3. **高杠杆地基项清单** —— 按出度 / ROI 排序；**显式标注"跨 N 条线共享"**；每项附"它解锁了什么、哪几条线同时受益"。
4. **分阶段攻坚路线** —— 每阶段：攻坚原子 + 完成判据；合法拓扑序；共享项前置。
5. **洞察** —— 零门槛起手项（今天就能练）/ 够不着、先放着的远期目标。

**不含**账本 / 预算 / 多线时间取舍 / 风险——那是可选进阶模块。

## 隐私铁律与范围闸

- 本 skill 是"方法"，不是任何人的数据。仓库内示例一律虚构脱敏；运行时产出的个人规划属用户私有，不随 skill 分发。
- 只产规划：**不替用户执行学习、不做进度追踪**。

## 进阶模块（可选，非默认）

仅当用户**明确要**"时间精力怎么排 / 钱怎么分 / 同时拼几件大事怎么取舍 / 风险点"时，读取并套用同目录 `进阶模块.md`；否则不加载、不产出这些内容。
