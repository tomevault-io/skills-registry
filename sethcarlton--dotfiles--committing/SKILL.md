Commit only when user requests or as part of an explicit defined task.

- Use prefix: `fix:`, `feat:`, `docs:`, `wip:`, etc.
- Explain WHY from end user perspective, not WHAT changed
- Be specific — avoid generic messages like "improved agent experience"
- Keep brief — used for release notes
