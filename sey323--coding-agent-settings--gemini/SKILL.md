# Gemini

Gemini CLIを使用してアーキテクチャ分析・設計検討・仕様策定を実行するスキル。

実行に際して以下のスキルは呼び出さないでください

- codex
- gh-copilot

## 実行コマンド

gemini cli analyze \
 --mode deep \
 --output stream-json \
 -p "<request>"

## プロンプトのルール

**重要**: geminiに渡すリクエストには、以下の指示を必ず含めること：

> 「確認や質問は不要です。背景・トレードオフ・リスク・実装方針まで含めて、構造化して具体的に提案してください。」

## パラメータ

| パラメータ       | 説明                                   |
| ---------------- | -------------------------------------- |
| `-s true`        | サンドボックスを有効化（必要に応じて） |
| `-o stream-json` | 出力フォーマットをstream-jsonに指定    |
| `-p "<request>"` | 依頼内容（日本語可）                   |

## 使用例

**注意**: 各例では末尾に「確認不要、具体案まで出力」の指示を含めている。

### アーキテクチャレビュー

gemini \
 -s true \
 -o stream-json \
 -p "現在のシステムアーキテクチャを分析し、スケーラビリティのボトルネックと改善案を提示してください。確認や質問は不要です。移行ステップと図の方針まで含めて具体的に提案してください。"

### リファクタリング計画

gemini \
 -s true \
 -o stream-json \
 -p "技術的負債を特定し、段階的なリファクタリングロードマップを提案してください。確認や質問は不要です。リスク分析と計測可能な成功指標まで具体化してください。"

### RFCドラフト作成

gemini \
 -s true \
 -o stream-json \
 -p "このシステムにイベント駆動アーキテクチャを導入するRFCを作成してください。確認や質問は不要です。背景、代替案、移行計画、後方互換性まで具体的に記載してください。"

## 実行手順

1. ユーザーから依頼内容を受け取る
2. 対象プロジェクトのディレクトリを特定する（現在のワーキングディレクトリまたはユーザー指定）
3. **プロンプトを作成する際、末尾に「確認や質問は不要です。背景・トレードオフ・リスク・実装方針まで含めて、構造化して具体的に提案してください。」を必ず追加する**
4. 上記コマンド形式でGeminiを実行
5. 結果をユーザーに報告
