To get code review notes on your work:

1. Navigate to either the `/front` or `/back` folder (NOT THE ROOT, WHICH DOESN'T HAVE A REPO)
2. Use `gh pr view --comments` on the current branch to read PR comments from CodeRabbit
3. Review each comment and autonomosly decide if it needs a fix or can be dismissed
4. Commit and push any diffs you make to the same branch
5. Outline all review notes and your decisions/actions on each

Notes:
- You should only ask for user permission if necessary
- Think carefully about whether recommendations are relevant or not
- Ensure your git commit message is descriptive
