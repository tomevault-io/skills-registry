# Talk Game Dot Design

이 스킬은 토크 게임의 도트 그래픽을 만들거나 수정할 때 적용한다.

## 빠른 시작

1. 자산 유형을 선택한다: `portrait`, `emotion`, `ui-icon`, `tile`, `prop`.
2. `references/asset-specs.md`에서 캔버스/앵커 규격을 확인한다.
3. 16색 팔레트(기본 12 + 강조 4)로 시안을 제작한다.
4. `references/review-checklist.md`로 품질을 검수한다.
5. 파일명/내보내기 규칙에 맞춰 정리한다.

## 핵심 규칙

1. **읽힘 우선**: 대사창 옆 2x 확대 기준에서 표정과 실루엣이 즉시 구분되게 만든다.
2. **해상도 고정**: 자산 타입별 캔버스를 임의 변경하지 않는다. 변경 필요 시 먼저 문서에 기록한다.
3. **팔레트 제한**: 한 자산에서 16색 초과를 금지한다. 색 추가보다 명도 대비로 해결한다.
4. **라인 최소화**: 외곽선은 필요한 부위만 사용하고, 내부선은 면 분할 목적일 때만 사용한다.
5. **애니메이션 경제성**: 감정 전달 목적이면 2~4프레임으로 해결한다.
6. **재사용 설계**: 머리/몸통/소품을 파츠로 분리해 감정 변형 비용을 줄인다.

## 작업 절차

### 1) 요구사항 확정

- 장면 맥락(감정, 거리감, 시간대)을 한 줄로 요약한다.
- 자산 유형과 목표 감정 1개를 먼저 확정한다.
- 불명확한 값은 `[TBD]`로 명시하고 추측 제작하지 않는다.

### 2) 실루엣 블로킹

- 1색으로 3안 이상 만든 뒤 식별성이 가장 높은 안을 채택한다.
- 축소(1x)와 확대(2x) 모두에서 머리-몸통-소품 구분을 확인한다.

### 3) 명암/색 배치

- `mid-tone`을 먼저 배치하고, `highlight`/`shadow`를 마지막에 얹는다.
- 같은 계열 색 이동은 hue shift(차가운 그림자, 따뜻한 하이라이트) 우선으로 처리한다.
- 고채도 색은 시선 유도 포인트(눈, 강조 UI) 외에는 제한한다.

### 4) 디테일/감정 보정

- 눈, 눈썹, 입 픽셀 이동만으로 감정 차이를 먼저 만든다.
- 노이즈성 텍스처(체크보드 남발)를 금지한다.
- 배경/프롭은 캐릭터 대비를 해치지 않게 채도·명도 우선순위를 낮춘다.

### 5) 검수 및 내보내기

- `references/review-checklist.md`를 모두 통과한 뒤 저장한다.
- 파일명은 `asset_<type>_<name>_<variant>_vNN.png` 형식을 사용한다.
- 스프라이트 시트는 프레임 간 간격 1px, 투명 배경 PNG로 내보낸다.

## 토크 게임 특화 패턴

### 패턴 1: 감정 프레임 세트

- `neutral`, `happy`, `angry`, `sad`, `surprised`를 기본 5종으로 유지한다.
- 감정별 차이는 눈/입/볼 강조 픽셀 중심으로 만들고 얼굴 비율은 고정한다.

### 패턴 2: 대사 UI 아이콘

- `16x16` 또는 `24x24` 기준을 유지한다.
- `1px` 스트로크 + 단색 그림자 1단계까지만 허용한다.
- 클릭/선택 상태는 색 변경보다 실루엣 변화(테두리, 마커)로 우선 표현한다.

### 패턴 3: 배경 타일

- `16x16` 기본 타일 + 변형 타일(코너/엣지)을 세트로 설계한다.
- 타일 경계 반복 노이즈를 방지하기 위해 3타일 루프 테스트를 필수로 수행한다.

## 참조 문서

- 자산 규격: `references/asset-specs.md`
- 최종 검수: `references/review-checklist.md`

## 주의사항

- 엔진 스케일(정수 배율)과 다른 비정수 리사이즈를 혼용하지 않는다.
- 안티앨리어싱 자동 필터를 켜고 저장하지 않는다.
- 동일 캐릭터의 팔레트/비율이 장면마다 바뀌지 않게 스타일 키를 유지한다.
