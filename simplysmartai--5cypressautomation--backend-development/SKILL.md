## Core Patterns for SMB Integrations

### 1. QuickBooks Online API Pattern
```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import httpx

class Invoice(BaseModel):
    customer_id: str
    line_items: list
    due_date: str

async def create_qbo_invoice(invoice: Invoice, access_token: str):
    """Create invoice in QuickBooks Online."""
    async with httpx.AsyncClient() as client:
        response = await client.post(
            f"{QBO_BASE_URL}/invoice",
            headers={"Authorization": f"Bearer {access_token}"},
            json=invoice.model_dump()
        )
        if response.status_code != 200:
            raise HTTPException(status_code=response.status_code)
        return response.json()
```

### 2. Webhook Handler Pattern
```python
@app.post("/webhooks/stripe")
async def handle_stripe_webhook(request: Request):
    payload = await request.body()
    sig = request.headers.get("Stripe-Signature")

    # Verify signature
    try:
        event = stripe.Webhook.construct_event(payload, sig, WEBHOOK_SECRET)
    except stripe.error.SignatureVerificationError:
        raise HTTPException(status_code=400, detail="Invalid signature")

    # Idempotent processing
    if await is_event_processed(event["id"]):
        return {"status": "already_processed"}

    # Handle event types
    match event["type"]:
        case "payment_intent.succeeded":
            await handle_payment_success(event["data"]["object"])
        case "invoice.paid":
            await sync_to_quickbooks(event["data"]["object"])

    await mark_event_processed(event["id"])
    return {"status": "processed"}
```

### 4. Resilient API Client
```python
from tenacity import retry, stop_after_attempt, wait_exponential

@retry(
    stop=stop_after_attempt(3),
    wait=wait_exponential(multiplier=1, min=4, max=10)
)
async def call_external_api(url: str, data: dict):
    """Call external API with automatic retry on failure."""
    async with httpx.AsyncClient(timeout=30.0) as client:
        response = await client.post(url, json=data)
        response.raise_for_status()
        return response.json()
```

## Architecture Principles

These aren't rules for their own sake — each one prevents a specific class of expensive bug:

1. **Validate at boundaries** — Use Pydantic for all external input. Bad data entering the system compounds: a malformed SKU in a form submission can ripple into a QBO validation error, then a shipping order failure, then a manual cleanup job. Validation at the edge stops that cascade at zero cost.

2. **Idempotent operations** — Use request IDs to prevent duplicates. Webhooks get delivered multiple times. Networks retry. Users double-click. If your invoice creation isn't idempotent, you'll eventually create a duplicate QBO invoice that needs manual credit memo cleanup.

3. **Async by default** — Use async/await for all I/O. External API calls (QBO, ShipStation, Stripe) can take 1-5 seconds each. Synchronous code blocks the event loop during that wait, making slow requests slow the entire server. Async handles concurrent requests efficiently without extra infrastructure.

4. **Structured logging** — Include correlation IDs in all logs. When a client says "invoice #1234 didn't create," you need to trace the full request path in seconds. Unstructured logs make this a 30-minute grep exercise. Structured logs make it a 10-second filter.

5. **Fail gracefully** — Return partial success where appropriate. If inventory check succeeds but QBO is down, return what you know rather than a generic 500. Clients can retry the specific failed step instead of restarting the entire workflow.
