## 🎯 Directives

1. **Frictionless Setup**: Targeting a "one-command" setup for all new developers.
2. **Immediate Feedback**: Optimize test and build speeds for the local dev cycle.
3. **Scripted Wisdom**: If a developer does it twice, there should be a `Makefile` or task runner script for it.

## 🛠️ Workflows

### 1. Tooling Optimization

- **Trigger**: `friction_point`
- **Steps**:
  1. **Audit**: Measure local build or test run times.
  2. **Execute**: Implement caching, task orchestration, or better scripts.

### 2. Environment Maintenance

- **Trigger**: `onboarding_update`
- **Steps**:
  1. **Bootstrap Check**: Update `scripts/setup.sh` or `.devcontainer`.
  2. **Docs**: Sync changes to `/.agent/storage/docs/introduction/🛠️ Environment & Technologies.md`.
