# Frontend Design Skill

## Core Philosophy
- **Mobile First**: Always design for small screens first, then scale up.
- **Visual Hierarchy**: Use spacing, size, and weight to guide the eye. Important things should be big/bold.
- **Simplicity**: A clean UI is better than a cluttered one.

## Implementation Rules
1. **CSS/Tailwind**: Use utility classes (Tailwind) if available. Group classes logically (layout -> spacing -> visual).
2. **Whitespace**: Give elements room to breathe. Use consistent padding/margins (multiples of 4px).
3. **Interactive States**: Every button and link MUST have `:hover`, `:active`, and `:focus` states.
4. **Consistency**: Re-use components (Buttons, Cards, Inputs) instead of hardcoding styles repeatedly.
