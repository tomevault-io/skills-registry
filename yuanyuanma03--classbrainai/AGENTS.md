## Project Overview

ClassBrainAI is a multi-agent collaboration system for intelligent class management, adapted from the edict architecture (which was based on the ancient Chinese "Three Provinces and Six Ministries" system). It uses a school class as the thematic metaphor, with AI agents representing various class committee positions (Monitor, Study Committee, Chair, Vice Monitor, etc.).

**Key Concept:** Tasks flow through a pipeline: Pending вЖТ Monitor вЖТ Study вЖТ Chair вЖТ Assigned вЖТ Doing вЖТ Review вЖТ Done

### State Machine

Task states follow a strict transition flow:
- `Pending` вЖТ `Monitor` вЖТ `Study` вЖТ `Chair` вЖТ `Assigned` вЖТ `Doing` вЖТ `Review` вЖТ `Done`
- `Blocked` can be entered from most states and can transition to `Doing` or `Cancelled`

See `_VALID_TRANSITIONS` in `scripts/class_board.py` for the complete rules.

### Data Storage

- **Primary data:** `data/tasks_source.json` - All task data
- **Derived data:** `data/live_status.json` - Generated by `refresh_live_data.py`
- **Concurrency:** Uses file locking (`scripts/file_lock.py`) for atomic operations

### Dashboard Architecture

- **Backend:** Python HTTP server using standard library (`dashboard/server.py`)
- **Frontend:** Single-file HTML/CSS/JavaScript (`dashboard/dashboard.html`)
- **API:** RESTful endpoints for live status, agent config, and task management

### File Lock Usage
Always use the atomic operations from `file_lock.py` when modifying `tasks_source.json`:
- `atomic_json_read()` - Read with shared lock
- `atomic_json_update()` - Read-modify-write with exclusive lock
- `atomic_json_write()` - Write with exclusive lock

## Development Notes

### When Modifying Agent Behavior
1. Edit the corresponding `SOUL.md` file in `agents/*/`
2. The SOUL.md defines the agent's personality and decision rules
3. Agents are invoked externally (not part of this codebase's runtime)

### When Adding New Task States
1. Update `STATES` list in `scripts/class_board.py`
2. Update `_VALID_TRANSITIONS` dict for state flow rules
3. Update `STATE_ORG_MAP` for agent mappings
4. Update frontend state displays in `dashboard/dashboard.html`

### When Modifying Dashboard UI
- The dashboard uses a campus color scheme (blue/cyan theme)
- CSS variables are defined in `<style>` section of `dashboard.html`
- State colors follow the pattern `.st-StateName`
